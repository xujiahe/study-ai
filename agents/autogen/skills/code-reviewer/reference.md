# 代码审查参考资料

## OWASP Top 10 安全风险

1. **注入攻击** — SQL、NoSQL、OS 命令注入
2. **身份验证失效** — 弱密码、会话管理缺陷
3. **敏感数据暴露** — 明文传输、弱加密
4. **XML 外部实体（XXE）** — 解析不受信任的 XML
5. **访问控制失效** — 越权访问
6. **安全配置错误** — 默认配置、不必要的功能开启
7. **XSS** — 反射型、存储型、DOM 型
8. **不安全的反序列化** — 远程代码执行风险
9. **使用含已知漏洞的组件** — 过时的依赖
10. **日志和监控不足** — 无法检测攻击

## 性能最佳实践

- 使用适当的数据结构（Map vs Object，Set vs Array）
- 避免在循环中进行 DOM 操作或 I/O
- 使用懒加载和分页处理大数据集
- 数据库查询添加索引
- 使用连接池而非每次新建连接
