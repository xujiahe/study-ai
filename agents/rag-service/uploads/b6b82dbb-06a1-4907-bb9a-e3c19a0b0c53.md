# 航点航线格式说明

航点航线使用 Template.kml（模板协议）与 Waylines.wpml（执飞协议）描述任务，二者打包在 KMZ 中。

## 协议概述

### Template.kml（模板协议）

用于定义任务的基本参数和规划区域，是用户在规划界面配置的原始数据，可被算法解析生成 Waylines.wpml 执飞文件。

### Waylines.wpml（执飞协议）

包含无人机实际执行飞行所需的全部信息，由 Template.kml 经过算法计算生成，包含具体的航点坐标和动作序列。

## 数据流

```
用户输入 → Template.kml（模板） → 算法计算 → Waylines.wpml（执飞） → 无人机执行
```

---

## Template.kml 协议

### XML 结构

```xml
<?xml version="1.0" encoding="UTF-8"?>
<kml xmlns="http://www.opengis.net/kml" xmlns:wpml="http://www.gdu.com/wpmz">
    <Document>
        <wpml:author></wpml:author>
        <wpml:createTime>0</wpml:createTime>
        <wpml:updateTime>0</wpml:updateTime>
        <wpml:version></wpml:version>
        <wpml:missionConfig>...</wpml:missionConfig>
        <wpml:folder>...</wpml:folder>
    </Document>
</kml>
```

### 根元素字段

| 字段名 | 类型 | 描述 | 示例值 |
|--------|------|------|--------|
| author | string | 作者信息 | "GDU" |
| createTime | uint64 | 创建时间戳（毫秒） | 1648456789000 |
| updateTime | uint64 | 更新时间戳（毫秒） | 1648456789000 |
| version | string | 协议版本号 | "1.0.0" |
| missionConfig | object | 任务配置 | - |
| folder | object | 模板信息 | - |

### MissionConfig（任务配置）

#### 基本信息

| 字段名 | 类型 | 单位 | 描述 | 取值范围 |
|--------|------|------|------|----------|
| name | string | - | 任务名称 | - |
| trackType | int32 | - | 航线类型 | 见下方枚举 |

#### trackType 枚举

| 值 | 说明 |
|----|------|
| 1 | 航点航线 |
| 2 | 正射影像（2D） |
| 3 | 四项倾斜 |
| 4 | 蛇形航线 |
| 5 | 带状航线 |
| 6 | 五向倾斜（3D） |
| 7 | 智能摆拍五向 |
| 8 | 智能摆拍三向 |
| 9 | 快速航线 |
| 10 | AI 巡检 |

#### 设备配置

| 字段名 | 类型 | 描述 |
|--------|------|------|
| drone | DroneInfo | 无人机信息 |
| payloads | PayloadInfo[] | 载荷信息列表 |
| globalPayloadLensIndex | int32 | 全局载荷镜头索引 |

##### DroneInfo

| 字段名 | 类型 | 描述 |
|--------|------|------|
| droneEnumValue | int32 | 无人机类型枚举值 |
| droneSubEnumValue | int32 | 无人机子类型枚举值 |

##### PayloadInfo

| 字段名 | 类型 | 描述 |
|--------|------|------|
| payloadPositionIndex | int32 | 载荷位置索引（0, 1, 2） |
| payloadEnumValue | int32 | 载荷类型枚举值 |
| payloadSubEnumValue | int32 | 载荷子类型枚举值 |

#### 飞行控制

| 字段名 | 类型 | 单位 | 描述 | 取值范围 |
|--------|------|------|------|----------|
| flyToWaylineMode | string | - | 飞向航线模式 | "safely", "directly" |
| finishAction | string | - | 完成动作 | 见下方枚举 |
| exitOnRCLost | string | - | 遥控器丢失行为 | "goContinue", "executeLostAction" |
| executeRCLostAction | string | - | 失控执行动作 | "goBack", "hover" |
| turnType | int32 | - | 到达航点高度方式 | 0=斜线, 1=折线 |
| continueFly | int32 | - | 是否继续飞行 | 0=否, 1=是 |

##### flyToWaylineMode 枚举

- `safely` - 安全模式（绕障飞行）
- `directly` - 直接模式（直线飞行）

##### finishAction 枚举

- `goHome` - 返航到起飞点
- `noAction` - 悬停在最后一个航点
- `jumpFly` - 机库跳飞
- `safeGoHome` - 安全点返航
- `alongGoHome` - 沿航线返航

#### 高度和速度

| 字段名 | 类型 | 单位 | 描述 | 取值范围 |
|--------|------|------|------|----------|
| takeOffSecurityHeight | double | 米 | 起飞安全高度 | 1-500 |
| globalTransitionalSpeed | double | m/s | 全局过渡速度 | 1-15 |
| globalRTHHeight | int32 | 米 | 全局返航高度 | 0-500 |
| takeOffEllipsoidHeight | double | 米 | 起飞椭球高度 | 0-500 |
| takeOffRefPointAGLHeight | int32 | 米 | 起飞点实际海拔高度 | 0-500 |

#### 起飞点

| 字段名 | 类型 | 描述 | 格式 |
|--------|------|------|------|
| takeOffRefPoint | string | 起飞参考点坐标 | "经度,纬度" |

#### 自动绕障

| 字段名 | 类型 | 描述 | 取值 |
|--------|------|------|------|
| autoReroute.missionAutoRerouteMode | int32 | 任务航线绕行模式 | 0=关闭, 1=开启 |
| autoReroute.transitionalAutoRerouteMode | int32 | 过渡航线绕行模式 | 0=关闭, 1=开启 |

### Folder（模板信息）

#### 基本属性

| 字段名 | 类型 | 单位 | 描述 | 取值范围 |
|--------|------|------|------|----------|
| templateType | string | - | 模板类型 | 见下方说明 |
| templateId | int32 | - | 模板 ID | 从 1 开始 |
| autoFlightSpeed | double | m/s | 自动飞行速度 | 1-15 |
| ellipsoidHeight | double | 米 | 全局椭球高 | 0-500 |
| globalHeight | double | 米 | 全局相对高度 | 0-500 |

#### templateType 枚举

| 值 | 说明 | 使用场景 |
|----|------|----------|
| waypoint | 航点航线 | 自定义航点飞行 |
| mapping2d | 2D 正射 | 正射影像测绘 |
| mapping3d | 3D 倾斜 | 倾斜摄影建模 |
| mappingstrip | 航带飞行 | 线状区域测绘 |
| targetdetection | AI 巡逻 | 智能巡检 |

#### 坐标系参数

| 字段名 | 类型 | 描述 | 取值范围 |
|--------|------|------|----------|
| coordinateSysParam.coordinateMode | string | 坐标模式 | "WGS84" |
| coordinateSysParam.heightMode | string | 高度模式 | 见下方枚举 |
| coordinateSysParam.positioningType | string | 定位类型 | "GPS", "RTK" |
| coordinateSysParam.globalShootHeight | double | 全局拍摄高度（米） | 0-500 |

##### heightMode 枚举

- `relativeToStartPoint` - 相对起飞点高度
- `WGS84` - 椭球高度模式（WGS84）
- `altitudeHeight` - 海拔高度
- `EGM96` - 大疆使用的海拔高编辑（EGM96）
- `realTimeFollowSurface` - 相对地形高度（仿地飞行）

#### 载荷参数

| 字段名 | 类型 | 描述 | 取值范围 |
|--------|------|------|----------|
| payloadParam.payloadPositionIndex | int32 | 载荷位置索引 | 0, 1, 2 |
| payloadParam.imageFormat | string | 图像格式 | "visible", "ir", "wide", "zoom" |

### Waypoint（航点航线）

当 `templateType="waypoint"` 时使用。

#### 全局参数

| 字段名 | 类型 | 单位 | 描述 | 取值范围 |
|--------|------|------|------|----------|
| gimbalPitchMode | string | - | 云台俯仰模式 | "manual", "usePointSetting", "fixed", "custom" |
| globalWaypointTurnMode | string | - | 全局航点转弯模式 | 见下方枚举 |
| globalGimbalPitchAngle | double | 度 | 全局云台俯仰角 | -90 ~ 30 |
| globalGimbalYawAngle | double | 度 | 全局云台方位角 | -180 ~ 180 |

##### globalWaypointTurnMode 枚举

- `coordinateTurn` - 协调转弯，不过点，提前转弯
- `toPointAndStopWithDiscontinuityCurvature` - 直线飞行，到点停
- `toPointAndStopWithContinuityCurvature` - 曲线飞行，到点停
- `toPointAndPassWithContinuityCurvature` - 协调转弯，过点转

#### 全局航向参数

| 字段名 | 类型 | 单位 | 描述 | 取值范围 |
|--------|------|------|------|----------|
| globalWaypointHeadingParam.waypointHeadingMode | string | - | 航向模式 | 见下方枚举 |
| globalWaypointHeadingParam.waypointHeadingAngle | double | 度 | 航向角度 | -330 ~ 330 |

##### waypointHeadingMode 枚举

- `followWayline` - 沿航线方向
- `manually` - 手动控制
- `fixed` - 固定角度
- `smoothTransition` - 平滑过渡
- `towardPOI` - 朝向兴趣点

#### Placemark（航点）

| 字段名 | 类型 | 单位 | 描述 |
|--------|------|------|------|
| index | int32 | - | 航点索引（从 0 开始） |
| ellipsoidHeight | double | 米 | 椭球高度 |
| height | double | 米 | 相对高度 |
| waypointSpeed | double | m/s | 航点速度 |
| gimbalPitchAngle | double | 度 | 云台俯仰角 |
| gimbalYawAngle | double | 度 | 云台方位角 |
| point.coordinates.coordinate | string | - | 坐标："经度,纬度,高度" |
| isRisky | int32 | - | 是否危险点（0=否, 1=是） |

#### ActionGroup（动作组）

| 字段名 | 类型 | 描述 |
|--------|------|------|
| actionGroupId | int32 | 动作组 ID |
| actionGroupStartIndex | int32 | 起始航点索引 |
| actionGroupEndIndex | int32 | 结束航点索引 |
| actionGroupMode | string | 执行模式："sequence"（串行）, "parallel"（并行） |
| actionTrigger | ActionTrigger | 触发器 |
| actions | Action[] | 动作列表 |

##### ActionTrigger（触发器）

| 字段名 | 类型 | 描述 | 取值 |
|--------|------|------|------|
| actionTriggerType | string | 触发类型 | 见下方枚举 |
| actionTriggerParam | double | 触发参数 | - |

###### actionTriggerType 枚举

- `reachPoint` - 到达航点时执行
- `betweenAdjacentPoints` - 航段触发，均匀转云台
- `multipleTiming` - 等时触发
- `multipleDistance` - 等距触发
- `betweenAdjacentPointsIncludeFirstPoint` - 航段触发，均匀转云台（含首航点）

##### Action（动作）

| 字段名 | 类型 | 描述 |
|--------|------|------|
| actionId | int32 | 动作 ID |
| actionActuatorFunc | string | 动作类型 |
| actionActuatorFuncParam | object | 动作参数 |

###### 常用动作类型

| 动作名称 | 说明 | 参数 |
|---------|------|------|
| takePhoto | 拍照 | takePhotoParam |
| startRecord | 开始录像 | startRecordParam |
| stopRecord | 停止录像 | - |
| hover | 悬停 | hoverParam.hoverTime（秒） |
| gimbalRotate | 云台旋转 | gimbalRotateParam |
| rotateYaw | 飞行器偏航 | rotateYawParam |
| zoom | 相机变焦 | zoomParam |

## Waylines.wpml 协议

### XML 结构

```xml
<?xml version="1.0" encoding="UTF-8"?>
<wpml xmlns="http://www.gdu.com/wpmz">
    <version></version>
    <mission>...</mission>
    <folders>...</folders>
</wpml>
```

### 根元素

| 字段名 | 类型 | 描述 |
|--------|------|------|
| version | string | 协议版本号 |
| mission | MissionConfig | 任务配置 |
| folders | Folder[] | 航线文件夹数组 |

### MissionConfig（任务配置）

与 Template.kml 中的 MissionConfig 结构相同，但不包含 name 和 trackType 字段。

| 字段名 | 类型 | 单位 | 描述 | 取值范围 |
|--------|------|------|------|----------|
| drone | DroneInfo | - | 无人机信息 | - |
| payloads | PayloadInfo[] | - | 载荷信息数组 | - |
| flyToWaylineMode | string | - | 飞向航线模式 | "safely", "directly" |
| finishAction | string | - | 完成动作 | "goHome", "noAction", "jumpFly", "safeGoHome", "alongGoHome" |
| exitOnRCLost | string | - | 遥控器丢失行为 | "goContinue", "executeLostAction" |
| executeRCLostAction | string | - | 失控执行动作 | "goBack", "hover" |
| takeOffSecurityHeight | double | 米 | 起飞安全高度 | 1-500 |
| globalTransitionalSpeed | double | m/s | 全局过渡速度 | 1-15 |
| globalRTHHeight | int32 | 米 | 全局返航高度 | 0-500 |
| turnType | int32 | - | 转弯类型 | 0=斜线, 1=折线 |
| gimbalPitchMode | string | - | 云台俯仰模式 | "manual", "usePointSetting", "fixed", "custom" |
| autoReroute | AutoRerouteInfo | - | 自动绕障配置 | - |

### Folder（航线文件夹）

| 字段名 | 类型 | 单位 | 描述 |
|--------|------|------|------|
| templateId | int32 | - | 模板 ID |
| executeHeightMode | string | - | 执行高度模式 |
| waylineId | int32 | - | 航线 ID |
| autoFlightSpeed | double | m/s | 自动飞行速度 |
| placemarks | Placemark[] | - | 航点数组 |

#### executeHeightMode 枚举

- `relativeToStartPoint` - 相对起飞点高度
- `WGS84` - EGM96 海拔高度

### Placemark（航点）

| 字段名 | 类型 | 单位 | 描述 | 取值范围 |
|--------|------|------|------|----------|
| index | int32 | - | 航点索引 | 从 0 开始 |
| executeHeight | double | 米 | 执行高度 | -1500 ~ 1500 |
| waypointSpeed | int32 | m/s | 航点速度 | 1-15 |
| point.coordinates.coordinate | string | - | 坐标 | "经度,纬度,高度" |
| waypointHeadingParam | WaypointHeadingParam | - | 航向参数 | - |
| waypointTurnParam | WaypointTurnParam | - | 转弯参数 | - |
| actionGroups | ActionGroup[] | - | 动作组列表 | - |
| isRisky | int32 | - | 是否危险点 | 0=否, 1=是 |

ActionGroup 和 Action 结构与 Template.kml 中相同。

## 使用示例

### Template.kml 示例（航点航线）

```xml
<?xml version="1.0" encoding="UTF-8"?>
<kml xmlns="http://www.opengis.net/kml" xmlns:wpml="http://www.gdu.com/wpmz">
    <Document>
        <wpml:author>GDU</wpml:author>
        <wpml:createTime>1648456789000</wpml:createTime>
        <wpml:updateTime>1648456789000</wpml:updateTime>
        <wpml:version>1.0.0</wpml:version>
        
        <wpml:missionConfig>
            <wpml:name>测试任务</wpml:name>
            <wpml:trackType>1</wpml:trackType>
            <wpml:flyToWaylineMode>safely</wpml:flyToWaylineMode>
            <wpml:finishAction>goHome</wpml:finishAction>
            <wpml:takeOffSecurityHeight>20</wpml:takeOffSecurityHeight>
            <wpml:globalTransitionalSpeed>15</wpml:globalTransitionalSpeed>
            
            <wpml:drone>
                <wpml:droneEnumValue>67</wpml:droneEnumValue>
                <wpml:droneSubEnumValue>1</wpml:droneSubEnumValue>
            </wpml:drone>
            
            <wpml:payloads>
                <wpml:payloadPositionIndex>0</wpml:payloadPositionIndex>
                <wpml:payloadEnumValue>53</wpml:payloadEnumValue>
                <wpml:payloadSubEnumValue>1</wpml:payloadSubEnumValue>
            </wpml:payloads>
        </wpml:missionConfig>
        
        <wpml:folder>
            <wpml:templateType>waypoint</wpml:templateType>
            <wpml:templateId>1</wpml:templateId>
            <wpml:autoFlightSpeed>10</wpml:autoFlightSpeed>
            <wpml:globalHeight>100</wpml:globalHeight>
            
            <wpml:waypoint>
                <wpml:globalGimbalPitchAngle>-90</wpml:globalGimbalPitchAngle>
                <wpml:placemarks>
                    <Placemark>
                        <wpml:index>0</wpml:index>
                        <wpml:height>100</wpml:height>
                        <wpml:waypointSpeed>10</wpml:waypointSpeed>
                        <Point>
                            <coordinates>114.5788605,30.4990137,100</coordinates>
                        </Point>
                        <wpml:actionGroups>
                            <wpml:actionGroupId>0</wpml:actionGroupId>
                            <wpml:actionGroupStartIndex>0</wpml:actionGroupStartIndex>
                            <wpml:actionGroupEndIndex>0</wpml:actionGroupEndIndex>
                            <wpml:actionGroupMode>sequence</wpml:actionGroupMode>
                            <wpml:actionTrigger>
                                <wpml:actionTriggerType>reachPoint</wpml:actionTriggerType>
                            </wpml:actionTrigger>
                            <wpml:actions>
                                <wpml:actionId>0</wpml:actionId>
                                <wpml:actionActuatorFunc>takePhoto</wpml:actionActuatorFunc>
                            </wpml:actions>
                        </wpml:actionGroups>
                    </Placemark>
                </wpml:placemarks>
            </wpml:waypoint>
        </wpml:folder>
    </Document>
</kml>
```

### Waylines.wpml 示例

```xml
<?xml version="1.0" encoding="UTF-8"?>
<wpml xmlns="http://www.gdu.com/wpmz">
    <version>1.0.0</version>
    
    <mission>
        <flyToWaylineMode>safely</flyToWaylineMode>
        <finishAction>goHome</finishAction>
        <takeOffSecurityHeight>20</takeOffSecurityHeight>
        <globalTransitionalSpeed>15</globalTransitionalSpeed>
        
        <drone>
            <droneEnumValue>67</droneEnumValue>
            <droneSubEnumValue>1</droneSubEnumValue>
        </drone>
        
        <payloads>
            <payloadPositionIndex>0</payloadPositionIndex>
            <payloadEnumValue>53</payloadEnumValue>
            <payloadSubEnumValue>1</payloadSubEnumValue>
        </payloads>
    </mission>
    
    <folders>
        <templateId>1</templateId>
        <executeHeightMode>relativeToStartPoint</executeHeightMode>
        <waylineId>1</waylineId>
        <autoFlightSpeed>10</autoFlightSpeed>
        
        <placemarks>
            <index>0</index>
            <executeHeight>100</executeHeight>
            <waypointSpeed>10</waypointSpeed>
            <point>
                <coordinates>114.5788605,30.4990137,100</coordinates>
            </point>
            <waypointHeadingParam>
                <waypointHeadingMode>followWayline</waypointHeadingMode>
            </waypointHeadingParam>
            <waypointTurnParam>
                <waypointTurnMode>coordinateTurn</waypointTurnMode>
            </waypointTurnParam>
            <actionGroups>
                <actionGroupId>0</actionGroupId>
                <actionGroupStartIndex>0</actionGroupStartIndex>
                <actionGroupEndIndex>0</actionGroupEndIndex>
                <actionGroupMode>sequence</actionGroupMode>
                <actionTrigger>
                    <actionTriggerType>reachPoint</actionTriggerType>
                </actionTrigger>
                <actions>
                    <actionId>0</actionId>
                    <actionActuatorFunc>takePhoto</actionActuatorFunc>
                </actions>
            </actionGroups>
            <isRisky>0</isRisky>
        </placemarks>
    </folders>
</wpml>
```

## 注意事项

1. 坐标格式统一为 WGS84，格式："经度,纬度,高度"
2. 角度单位为度，速度单位为米/秒，高度单位为米
3. Template.kml 是规划数据，Waylines.wpml 是执行数据
4. 云台俯仰角范围：-90°（垂直向下）~ 30°（向上）
5. 航向角特殊值 720 表示沿航线方向，对应 `WaypointHeadingMode.followWayline`（与机头角度数值模式区分）
6. 动作组支持串行和并行执行，合理使用可提高任务效率
7. 高度模式选择要根据地形特点，复杂地形建议使用仿地飞行
8. 重叠度设置影响成图质量，建议航向 70-80%，旁向 60-70%
